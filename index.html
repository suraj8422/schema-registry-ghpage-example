The type of the element in FED terms. Must be one of the below values- button is any clickable button. Note- there is significant business interest to track clicks to Play or Resume. tile is a clickable content tile, regardless of intent. containerType
string (ContainerType)

The type of container. Must be one of the below values- shelf is a horizontal-scrolling container that renders clickable content tiles. You cannot scroll past the end. carousel is the same a shelf, except scrolling past the end 'wraps you around' back to the 1st tile. grid is a vertical-scrolling container that renders clickable content tiles. It does not contain individual carousels. i.e. Movies/Series landing pages are grids. menu_list is a container that renders a list of clickable options i.e. nav bar, genre picker, play/add to watchlist buttons, suggested/extras/details buttons, suggested search terms. cta_button is a container that renders CTA buttons that allow a user to proceed and move forward from a page (i.e. buy now, agree & continue, confirming a premier access purchase, etc...) footer is a container that includes the various footer links found on all onboarding pages (i.e. Privacy Policy, Do Not Sell My Info, etc...) overlay is a container that loads over an existing pageView rather than having another pageView fire as a new screen (i.e. Purchase Confirmed screen on Web, Email Not Recognized Modal on Account Log In flow, New Email Sent on Password Reset flow, etc...) form is a container that renders various input-related items or buttons i.e. input_form for search or Onboarding, check box or toggle. nav_button is a container that contains the down arrow on the web MLP, splash, or the back button (left arrow) on IAP onboarding screens. mlp_log_in is the container on the MLP that captures the floating 'Log In' link on the upper right (which expands to 'Sign Up Now' as a user scrolls down the MLP) mlp_body is the container on the MLP that stretches from just below the down arrow to just above the footer links. This container is intended to allow us to capture each messaging as invisible elements for scroll depth tracking purposes. onboarding_paywall is a container on the paywall that includes payment type selection and on bundle flows, monthly and annual plan selection or the plan selection CTA buttons on IAP. onboarding_payment_info is a container on the paywall that includes the payment info fields on web. plan_picker is the container on the plan selection screen as part of the Pittsburgh Early Access MLP Purchase Flow. license_plate_code is the container holding the numerical code on the license plate screen of select connected devices for off-device sign up and log in flows. text is the container holding non clickable text in the app (i.e. content summaries or restricted text on details / collections pages) sca_modal is the container that contains the Strong Customer Authentication modal that appears after a user submits their payment info in EU countries. splash_log_in is the container on the splash page that captures the floating 'Log In' link on the upper right (which expands to 'Sign Up Now' as a user scrolls down the splash page) splash_body is the container on the splash page that stretches from just below the down arrow to just above the footer links. This container is intended to allow us to capture each messaging as invisible elements for scroll depth tracking purposes. hero is the new generic containerType that will be used for all containers with type of HeroContainer or HeroFullBleedContainer instead of carousel. containerStyle will be used to distinguish the types of hero. ledger is the new generic containerType that will be used for all containers that ask users to confirm billing details and agree to a specific subscription, whether it be a net new subscription or a change in subscription.

interactionType
string (InteractionType)

The description of the interaction. It must be one of the values defined below. Contact DGI if a new value must be added to pass validation. select on AAW (Apple, Android, Web) is the click of an Element. focus on CD (Connected Devices) is the focus on an Element. Doing a focus to an Element that is still a focus event (vs. a scroll-* event). scroll-* events are on AAW only and fire on the completion of the scroll action. If an element or container is scrolled past, once the scroll is completed, those elements or containers should be included in the payload. FED concern- tracking as each element comes into view is heavy (edge case- scrolling thru the entire Simpsons series [30+ seasons]). The payload for interaction scroll events should include all containerViewIds that were in view or scrolled by until the completion of the scroll action. deeplink refers to external links that land users in the app (such as a Video Player or Content Details screen) autoplay is when a Video Player automatically plays a video (dependent on video, how a user landed on the video, geo, etc...) play_button may trigger an alternate navigation path. As opposed to select, which may lead you to details. Clicking the play/pause button on a CD remote may lead you directly into playback. long_press may trigger an alternate navigation path. For example, on an episode cell it may bring you to the details page instead of directly into playback. two_finger_tap is when a user taps on a cell with two fingers instead of one. This can result in different behavor than select. For example, going directly to playback instead of details. guest_autoplay is when a guest in a groupwatch is automatically brought into the video player without selecting anything. For example, this can occur when the guest is waiting in the room for the stream to start or on the group watch play next episode screen. long_press_play_button is when a user long presses the play button (i.e. holds it down) on a bookmark tile (e.g. Continue Watching or an episode tile on details). This may trigger an alternate navigation path. This container is intended to allow us to capture each messaging as invisible elements for scroll depth tracking purposes. hero is the new generic containerType that will be used for all containers with type of HeroContainer or HeroFullBleedContainer instead of carousel. containerStyle will be used to distinguish the types of hero. ledger is the new generic containerType that will be used for all containers that ask users to confirm billing details and agree to a specific subscription, whether it be a net new subscription or a change in subscription.
